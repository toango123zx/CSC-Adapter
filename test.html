📐 IStandardMessage — Interface cốt lõi
Định dạng tin nhắn chung dùng xuyên suốt toàn hệ thống

Mọi tin nhắn từ bất kỳ nền tảng nào (Telegram, LiveChat...) đều được chuẩn hóa về dạng IStandardMessage trước khi đưa vào Chat Hub xử lý. Đây là "ngôn ngữ chung" giúp các adapter giao tiếp được với nhau.

Trường Kiểu Bắt buộc Mô tả
platform Platform enum YES Nền tảng gốc: TELEGRAM hoặc LIVECHAT
platformThreadId string YES ID cuộc hội thoại trên nền tảng gốc (Telegram: chatId, LiveChat: chatId)
platformMessageId string YES ID tin nhắn trên nền tảng gốc
senderType SenderType enum YES USER = khách hàng, AGENT = nhân viên, SYSTEM = hệ thống
senderName string YES Tên hiển thị của người gửi
senderId string? OPT ID người gửi trên nền tảng gốc
messageType MessageType enum YES TEXT / IMAGE / FILE / STICKER / LOCATION / CONTACT / SYSTEM_EVENT
text string? OPT Nội dung văn bản (có với TEXT, caption với IMAGE/FILE)
mediaUrl string? OPT URL file hoặc ảnh đính kèm
metadata Record<string, any>? OPT Dữ liệu bổ sung tùy nền tảng (width/height ảnh, fileName, latitude...)
timestamp Date YES Thời điểm gửi tin nhắn
⚙️ Cấu hình môi trường (.env)
Các biến cần thiết để chạy ứng dụng
Biến Ví dụ Mô tả
PORT 3000 Port server lắng nghe
TELEGRAM_BOT_TOKEN 1234567890:AAF... Token bot từ @BotFather trên Telegram
LIVECHAT_ACCOUNT_ID abc-123... Account ID LiveChat (dạng UUID)

Chat Hub API

Điều phối tin nhắn và quản lý adapter — /chat/*

POST /chat/send Gửi tin nhắn đến khách hàng qua API ▼

Gửi tin nhắn trực tiếp đến một khách hàng cụ thể trên bất kỳ nền tảng nào (Telegram hoặc LiveChat) mà không cần qua luồng routing tự động. Thường dùng để hệ thống CRM hoặc backend tự động push thông báo (ví dụ: "Đơn hàng #123 đã được giao thành công!").

1
Nhận DTO từ request body
→
2
Lấy adapter theo platform
→
3
Tạo IStandardMessage với senderType = AGENT
→
4
Gọi adapter.sendMessage()
→
5
Trả về { success, message }
Request Body (JSON)
Tham sốKiểuBắt buộcMô tả & Ví dụ
platform "TELEGRAM" | "LIVECHAT" YES Nền tảng đích cần gửi tin. "TELEGRAM" sẽ gửi qua Telegram Bot, "LIVECHAT" push qua WebSocket đến Dashboard nhân viên
threadId string YES ID cuộc hội thoại. Với Telegram: là Chat ID (số nguyên dạng chuỗi, VD: "123456789"). Với LiveChat: là chat_id
text string YES Nội dung tin nhắn. VD: "Đơn hàng của bạn đã được giao thành công!"
senderName string? OPT Tên hiển thị người gửi. Mặc định: "System API". Hiển thị dạng: 💬 [Hệ thống CRM]: Nội dung...
messageType "TEXT" | "IMAGE" | "FILE" OPT Loại tin nhắn. Mặc định: "TEXT"
mediaUrl string? OPT URL ảnh/file nếu messageType là IMAGE hoặc FILE
Ví dụ Request
JSON — POST /chat/send 
{
  "platform": "TELEGRAM",
  "threadId": "123456789",       // Telegram Chat ID của khách
  "text": "Đơn hàng #ORD-001 đã được giao thành công!",
  "senderName": "Hệ thống CRM",
  "messageType": "TEXT"
}
Response
201 Thành công
JSON — Response 201
{
  "success": true,
  "message": "Đã gửi tin nhắn"
}
400 Lỗi validation DTO (thiếu trường bắt buộc, sai kiểu dữ liệu)
GET /chat/adapters Xem danh sách adapter đã đăng ký ▼

Kiểm tra xem những adapter nào đã sẵn sàng hoạt động. Khi ứng dụng khởi động, TelegramService và LivechatGateway tự động đăng ký với ChatHubService.

JSON — Response 200

  "adapters": ["TELEGRAM", "LIVECHAT"]

Telegram API

Quản lý Telegram Bot — /telegram/*

ℹ️
Telegram Bot sử dụng Long Polling (tự kết nối đến server Telegram) — không cần cấu hình webhook. Bot tự động xử lý: /start, /help, /info, tin nhắn text, ảnh, file, sticker, location, contact.
GET /telegram/bot-info Xem thông tin Bot ▼

Gọi getMe() của Telegram API để lấy thông tin bot đang chạy. Dùng để xác nhận bot đã kết nối thành công với token đã cấu hình.

  }
}

Gửi tin nhắn text thẳng đến một Chat ID Telegram, bỏ qua Chat Hub. Dùng cho testing, thông báo hệ thống đơn giản. Khác với /chat/send — endpoint này không tạo IStandardMessage, gọi thẳng Telegram Bot API.

Tham sốKiểuBắt buộcMô tả

text string YES Nội dung tin nhắn text cần gửi
Ví dụ Request
JSON
{
  "chatId": "123456789",
  "text": "Xin chào! Đây là thông báo từ hệ thống."

JSON — Response 201
{

  "message": "Đã gửi tin nhắn"

POST /telegram/send-photo Gửi ảnh trực tiếp ▼

Gửi ảnh (có thể kèm caption) trực tiếp đến một Chat ID Telegram, bỏ qua Chat Hub. Ảnh phải là URL công khai truy cập được.

Request Body
Tham sốKiểuBắt buộcMô tả
chatId string YES Chat ID Telegram người nhận
photoUrl string YES URL ảnh công khai. VD: "https://example.com/product.jpg"
caption string? OPT Chú thích dưới ảnh
🤖 Telegram Bot — Lệnh & Sự kiện tự động xử lý
Bot lắng nghe và xử lý các loại tin nhắn sau từ khách hàng
LoạiTriggerHành độngIStandardMessage.messageType
/start Lần đầu dùng bot Gửi tin nhắn chào mừng, hướng dẫn. Không route đến Hub —
/help Khách gõ /help Gửi hướng dẫn sử dụng —
/info Khách gõ /info Hiển thị User ID, Chat ID — quan trọng để lấy chatId —
text Tin nhắn văn bản Parse → Route đến Hub → Forward đến LiveChat agents TEXT
photo Gửi ảnh Lấy ảnh resolution cao nhất, lấy fileLink → Route đến Hub IMAGE
document Gửi file Lấy fileLink, fileName, mimeType → Route đến Hub FILE
sticker Gửi sticker Lưu emoji + setName vào metadata → Route đến Hub STICKER
location Chia sẻ vị trí Lưu lat/lng vào metadata, text = "📍 Vị trí: lat, lng" LOCATION
contact Chia sẻ liên hệ Lưu phone, firstName, lastName → Route đến Hub CONTACT

LiveChat API

Toàn bộ tích hợp LiveChat Agent API v3.5 — /livechat/*

ℹ️
Tất cả các API LiveChat đều gọi đến https://api.livechatinc.com/v3.5/agent/action/{action} bằng HTTP POST với header Authorization: Basic {base64_token}. Method đều là POST — kể cả khi "lấy danh sách".
GET /livechat/test Kiểm tra kết nối LiveChat ▼

Thử gọi list_chats để kiểm tra token và kết nối đến LiveChat API có hoạt động không. Là bước đầu tiên nên chạy sau khi cấu hình .env.

Response thành công
JSON — 200 OK
{
  "success": true,
  "message": "Kết nối LiveChat thành công!",
  "data": { ... }  // Dữ liệu chats trả về từ LiveChat API
}
Response thất bại
JSON — 200 (success: false)
{
  "success": false,
  "message": "Lỗi: Authentication failed"
}

POST /livechat/chat/:chatId/resume Tiếp tục (resume) chat đã đóng ▼

Mở lại một cuộc chat đã bị deactivate. Gọi LiveChat action resume_chat. Tạo thread mới trong chat cũ.

URL Params
Tham sốKiểuMô tả

POST /livechat/chat/:chatId/deactivate Đóng chat ▼

Kết thúc cuộc chat (đóng thread hiện tại). Gọi LiveChat action deactivate_chat. Chat vẫn còn trong hệ thống nhưng không active nữa.

POST /livechat/chat/:chatId/follow Follow / Unfollow chat ▼

Follow chat (follow_chat): Đăng ký nhận thông báo khi có sự kiện trong chat này. Agent phải follow mới nhận được events qua WebSocket từ LiveChat.

Unfollow chat (unfollow_chat): Hủy nhận thông báo. URL: POST /livechat/chat/:chatId/unfollow

🔔 Webhooks
⚠️
Cơ chế Webhook LiveChat: Khi có sự kiện (tin nhắn mới, chat bắt đầu...) trên LiveChat, LiveChat sẽ tự động gửi HTTP POST đến URL bạn đăng ký. Endpoint nhận là POST /livechat/webhook/incoming. URL này phải được exposed ra internet (dùng ngrok khi dev local).
GET /livechat/webhooks Danh sách webhooks đã đăng ký ▼

Lấy danh sách các webhook đang active trên tài khoản LiveChat. Gọi action list_webhooks.

POST /livechat/webhook/register Đăng ký webhook mới ▼

Đăng ký một URL để nhận thông báo khi có sự kiện cụ thể xảy ra trên LiveChat. Gọi action register_webhook.

Request Body
Tham sốKiểuBắt buộcMô tả
url string YES URL nhận webhook. VD: "https://yourdomain.com/livechat/webhook/incoming"
action string YES Sự kiện cần lắng nghe. VD: "incoming_chat", "incoming_event", "chat_deactivated"
secretKey string YES Secret key dùng để verify webhook. LiveChat gửi signature trong header X-LiveChat-Signature
Ví dụ Request

const socket = io("http://localhost:3000/livechat");

// Lắng nghe tin nhắn từ khách hàng
socket.on("new_message_to_agent", (data) => {
  console.log(`Khách ${data.messageData.senderName}: ${data.messageData.text}`);
  // → Hiển thị lên giao diện chat agent
});

// Nhân viên trả lời

  customerThreadId: "123456789",  // Telegram Chat ID khách
  agentName: "Nguyễn Văn A",
  text: "Chào bạn! Tôi có thể hỗ trợ gì cho bạn?"
});

WS livechat_webhook Forward webhook từ LiveChat vào Gateway ▼

Khi nhận được webhook từ LiveChat (qua REST endpoint), frontend hoặc proxy có thể forward payload này vào Gateway qua WebSocket để broadcast cho tất cả agents đang kết nối. Gateway sẽ phân tích action và phát event new_message_to_agent nếu là tin nhắn mới.
